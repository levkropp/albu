Option A: Albu-4 Custom 4-bit CPU

A custom-designed 4-bit processor using approximately 900–1,000 NMOS transistors. Simple enough to be feasible at moderate yield, complex enough to be a real programmable computer.

Why 4-bit?

3-bit saves only ~130 transistors but loses BCD arithmetic and halves the instruction space. Below 4-bit is a programmable state machine, not a CPU. 4-bit is the minimum width that supports BCD (binary-coded decimal), 16 distinct instructions, and useful calculator-class arithmetic.

Runs a program that counts in binary on 4 LEDs, or adds two numbers entered via DIP switches.

Option B: SERV RISC-V Core

Based on the SERV project by Olof Kindgren — the world's smallest RISC-V CPU. A full 32-bit processor that processes one bit at a time, trading speed for minimal transistor count.

External Register File

Key insight: SERV is designed with an external register file. Only the CPU core logic needs to be on our chip. The 32×32-bit register file (which would add ~6,000 transistors) can be a $0.50 commodity SRAM IC. This cuts the on-chip transistor count nearly in half.

Why SERV Matters

Full RISC-V software compatibility. GCC toolchain, Zephyr RTOS. “Hello world” compiled with standard tools, running on a chip made from eggs. The software ecosystem is the payoff for the extra transistors.

Bit Width Analysis

Why 4-bit is the sweet spot for a custom design, and why SERV is worth the extra transistors if yield allows.

BitsTransistors (est.)Useful?
2~690Barely — programmable state machine
3~780Marginal — 8 instructions, values 0–7

8~1,440Classic — but 1.5x the transistors of 4-bit

For comparison: Intel 4004 had 2,300 PMOS transistors in 1971. Albu-4 targets ~910 NMOS transistors for comparable 4-bit functionality using modern logic minimization. SERV at ~4,500 NMOS transistors delivers a full 32-bit RISC-V — roughly 2x the 4004 for 8x the data width.

Honest Challenges

Phase 4 is the most ambitious goal in the Albu project. These are the hard problems that must be solved:

Yield

The fundamental challenge. Every defect source must be identified and eliminated: particles in the resist, pinholes in the gate dielectric, incomplete etching, contact resistance variation, film thickness non-uniformity. Statistical process control across hundreds to thousands of devices is required.

Alignment Tolerance

Layer-to-layer alignment must be better than ~1/3 of minimum feature size. The Phase 3 camera-based alignment system targets this, but thermal drift, stage hysteresis, and substrate warping can all degrade real-world performance.

Multi-Layer Process Complexity

A multi-layer process means the substrate goes through coating, exposure, development, and etching multiple times. Each cycle introduces opportunities for contamination, misalignment, and damage to previous layers. Process integration — ensuring each step is compatible with all prior steps — is a major engineering effort.

NMOS-Only Logic

Both CPU options use NMOS-only logic with resistive pull-ups, avoiding the complexity of matching NMOS and PMOS characteristics. This simplifies fabrication but increases static power consumption and limits speed. For a proof-of-concept CPU, this trade-off is acceptable.

Interconnect Routing

Even 910 transistors need to be wired together. At 10–25μm feature size, two metal layers may not provide enough routing density. Via reliability (electrical connection between Metal 1 and Metal 2) is critical — a single open via can kill a signal path.

The honest truth: Phase 4 may take years of iteration. The intermediate milestones (inverter, ring oscillator, flip-flop, adder) are valuable demonstrations in their own right. Even a working 4-bit adder fabricated at home from grocery store materials would be a significant achievement. The full CPU is the north star, not a short-term deliverable.
