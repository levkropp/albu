Phase 3: Automated Chip Printer

Machine 3 — a maskless direct-write lithography system with machine vision alignment. Software-defined patterns, sub-micron alignment, and 10μm feature resolution from off-the-shelf components.

Machine Architecture

The chip printer is a precision XY scanning system that focuses a single-mode laser through a microscope objective to directly write patterns onto photoresist-coated substrates. A beam splitter enables through-the-lens (TTL) camera alignment, so the same optical path used for exposure is also used for finding alignment marks.

Core Components

  • Laser source: Single-mode 405nm diode (or 275nm UV-C LED for higher resolution). Focused through microscope objective to a diffraction-limited spot.
  • Microscope objective: 20x or 50x long-working-distance (LWD) objective. Defines the exposure spot size and field of view.
  • Beam splitter: 50/50 plate or cube splitter. Sends laser down through the objective and reflects the sample image up to the camera.
  • Machine vision camera: 5MP+ monochrome camera on the tube lens port. Provides live view of the substrate for alignment and focus.
  • Precision XY stage: Closed-loop linear stages with optical encoders. ~1μm positioning accuracy. The substrate moves; the optics stay fixed.
  • Z focus stage: Motorized or piezo-driven for autofocus. Maintains the laser spot at the resist surface across the entire die.
  • Motion controller: Klipper firmware running on a BTT Octopus board with TMC2209 stepper drivers. Proven in 3D printers, repurposed for nanometer-class motion control.
  • Software: Python + OpenCV for alignment mark detection, GDS-to-scan-path conversion, and exposure sequencing.

Why Maskless Over Photomask

Traditional photolithography projects a pattern through a physical mask. Maskless direct-write eliminates the mask entirely.

AdvantageExplanation
No mask fabricationNo need to produce a chrome-on-glass mask for each layer. Saves cost and turnaround time.
Software alignmentLayer-to-layer alignment is computed from detected alignment marks, not from mechanical pin registration.
Instant pattern changesModify the GDS file and re-expose. No physical artifact to remake.
Per-layer transformEach layer's coordinate system is computed from actual alignment mark positions, correcting for substrate rotation and translation.

The pipeline: GDS file → scan coordinates → affine transform → exposure. The entire pattern is defined in software.

Backup approach: If direct-write resolution proves insufficient, a 10x photomask reduction system (mask printed at 10x scale, projected through a reduction objective) provides an alternative path to 10μm features.

Software Alignment

Multi-layer chip fabrication requires each layer to be precisely aligned to the previous one. Machine 3 uses computer vision to detect alignment marks and compute an affine transform for each exposure layer.

Alignment Procedure

  1. Find alignment mark #1. Stage moves to the expected location. Camera acquires image. OpenCV template matching or contour detection locates the mark center. Record actual position (x1, y1).
  2. Find alignment mark #2. Stage moves to the second mark. Same detection. Record actual position (x2, y2).
  3. Compute affine transform. From the two measured positions vs. the two expected positions, compute the translation, rotation, and scale that maps design coordinates to stage coordinates.
  4. Apply transform. Every exposure coordinate in the GDS scan path is transformed through this matrix before being sent to the stage controller.

Alignment Accuracy

The theoretical alignment precision is determined by the camera resolution divided by the objective magnification:

alignment_precision = camera_pixel_size / objective_magnification

Example: 2.2μm pixel (5MP sensor) / 20x objective = 110nm per pixel
With sub-pixel fitting (centroid): ~50nm alignment accuracy

At 10μm feature sizes, a 50nm alignment budget is more than sufficient. Even 1μm alignment (20x worse) would be acceptable.

Exposure Strategy: Step-and-Scan

The objective field of view limits how much area can be exposed at once. Machine 3 uses a step-and-scan strategy: expose one field, step to the next, repeat until the full die is covered.

ParameterValue
Objective20x LWD
Field of view~500μm × 500μm
Die size7mm × 7mm
Fields per die14 × 14 = 196
Raster step within field10μm
Time per field~1 second
Time per layer~3–4 minutes
Total for 8 layers~30 minutes

Target resolution: 10μm features. This is achievable with a 405nm laser through a 20x objective (diffraction limit ~0.5μm), leaving comfortable margin for mechanical and optical imperfections.
